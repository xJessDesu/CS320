How can I ensure that my code, program, or software is functional and secure?

 - To ensure that software is functional and secure proper testing is a great way to check. Through the use of JUnit tests, code can easily be used to test individual aspects of the code on execution. This can very quickly and efficiently find potential errors within your code with the proper coverage. You can use other forms as testing as well depending on what the software amd code needs. For instance, integration testing can be used for larger programs. A slower and more cost effective method that can also help with the security of the code would be manual testing. Implementing any of these strategies greatly increases the chances of functional well-written code. 

How do I interpret user needs and incorporate them into a program?

 - Once you have gathered user needs it is best to translate those needs into requirements for the program. You set those requirements as goals and build and test the program around those goals. This ensures that the code is funsctional and in line with what the customer is asking for. 

How do I approach designing software?

 - The Software Development Lifecycle can be approached by seperating it into key phases. First you plan the project out based on user requirements. Then you create a design or framework to build the code around. Next up, you actually develop the code. After the code is developed comes testing before finally delpoying the final product. Some customers or projects may require maintenance after deployment.
